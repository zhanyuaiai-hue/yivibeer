# 📝 文案配置系统使用指南

## 🎯 功能说明

我为你创建了一个**可视化文案配置面板**，让你可以：
- ✍️ 直接在页面上编辑所有文案
- 👀 实时预览效果
- 💾 保存修改到浏览器
- 📤 导出配置为JSON文件
- 🔄 随时重置为默认文案

## 🚀 如何使用

### 1. 打开配置面板

开发环境下，页面右侧会出现一个**紫色浮动按钮**（✍️图标），点击即可打开配置面板。

**注意**：仅在开发环境显示，生产环境自动隐藏。

### 2. 编辑文案

配置面板分为6个区域：
- 🌐 **网站基础** - 站点名称、标语、版权信息
- 🔐 **登录页面** - 登录相关的所有文案
- 🧭 **导航栏** - 导航链接和按钮文案
- 📖 **品鉴日记** - 日记页面的所有文案
- 📚 **酒神教义** - 指南页面的所有文案
- 🏆 **酣战榜** - 名人堂页面的所有文案

点击区域标题展开，直接修改文案即可。

### 3. 搜索功能

在顶部搜索框输入关键词，快速定位相关文案。

例如：
- 搜索 "登录" → 显示所有包含"登录"的文案
- 搜索 "button" → 显示所有按钮文案

### 4. 保存修改

- 点击底部**"✅ 保存所有修改"**按钮
- 页面会自动刷新，查看效果

### 5. 导出配置

点击顶部 **💾 图标**，下载配置文件 `yivi-copy-config.json`。

### 6. 重置文案

点击顶部 **🔄 图标**，恢复为默认文案。

## 📋 在代码中使用文案

### 方法1：使用 getCopy 函数

```vue
<script setup>
import { getCopy } from '@/config/copy'
</script>

<template>
  <h1>{{ getCopy('site.name') }}</h1>
  <p>{{ getCopy('site.tagline') }}</p>
  <button>{{ getCopy('diary.addButton') }}</button>
</template>
```

### 方法2：导入整个配置

```vue
<script setup>
import { copyConfig } from '@/config/copy'
</script>

<template>
  <h1>{{ copyConfig.site.name }}</h1>
  <p>{{ copyConfig.diary.title }}</p>
</template>
```

### 方法3：响应式使用（推荐）

```vue
<script setup>
import { ref, computed } from 'vue'
import { getCopy } from '@/config/copy'

// 创建响应式文案
const copy = computed(() => ({
  title: getCopy('diary.title'),
  subtitle: getCopy('diary.subtitle'),
  addButton: getCopy('diary.addButton')
}))
</script>

<template>
  <h1>{{ copy.title }}</h1>
  <p>{{ copy.subtitle }}</p>
  <button>{{ copy.addButton }}</button>
</template>
```

## 🎨 文案编辑技巧

### 标题文案
- 简洁有力，3-5个字最佳
- 例如："醉境" 比 "我的品鉴日记" 更有韵味

### 按钮文案
- 动词开头，明确动作
- 例如："记录新的一杯" 比 "添加" 更有感觉

### 提示文案
- 温和友好，帮助用户理解
- 例如："日记本还是空白的" 比 "没有记录" 更文艺

### 错误提示
- 清晰直接，告诉用户怎么做
- 例如："请至少填写啤酒名称" 比 "输入错误" 更有用

## 📁 文件位置

- **配置文件**：`src/config/copy.js`
- **面板组件**：`src/components/CopyPanel.vue`

## 🔧 高级用法

### 批量修改

1. 导出配置文件（💾）
2. 在文本编辑器中批量修改
3. 将修改后的配置复制回配置面板
4. 保存

### 多语言支持（扩展）

可以复制 `copy.js`，创建 `copy.en.js` 用于英文版本。

## ⚠️ 注意事项

1. **数据存储**：修改保存在浏览器 localStorage 中
2. **清除数据**：清除浏览器数据会丢失自定义文案
3. **备份建议**：定期导出配置文件备份
4. **生产环境**：面板仅在开发时显示，部署后自动隐藏
